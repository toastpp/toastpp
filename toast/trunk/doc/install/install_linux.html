Toast++ Linux installation

Prerequisites

Compiler

For compiling the Toast++ C++ libraries, you need the GNU compiler family, including gcc, g++ and gfortran, version 4.2 or later. Note that if you also want to compile the Matlab toolbox, you have to make sure that your version of gcc is compatible with your version of Matlab. The Mathworks website has a compatibility list for recent Matlab versions.

Matlab toolbox

For compiling the Matlab toolbox, you need the Matlab mex compiler for your Matlab version. See above for compatibility considerations. The toolbox has been tested with Matlab 2012b. Later versions may or may not work.

Python module

For compiling the Python Toast++ module, you need Python installed on your system. Toast++ has been tested with Python 2.7.

Documentation

If you want to compile the source-level documentation, you need doxygen (found at http://www.stack.nl/~dimitri/doxygen/).

External libraries

Toast++ can make use of optional external libraries for including additional functionality and improving performance. See below under Configuration for details.

Configuration

Before configuring the compilation scripts, you need to set the TOASTDIR environment variable:

cd <toast-root>
setenv TOASTDIR $PWD   # for csh, or
export TOASTDIR=$PWD   # for bash

Configure the makefiles. In the simplest case (without any additional options), just type

./configure

To display a list of available options, type

./configure --help

Common configuration options

  • Debug version: To compile toast with debugging symbols and additional run-time checks, add the following option to your configure command:

    --enable-debug

    Note that the debugging version will be slower.
  • Thread support: Toast++ uses the POSIX thread library (pthreads) to accelerate certain operations such as matrix assembly and solution on shared-memory platforms. To compile Toast++ with thread support, add the following option to your configure command:

    --enable-threads

  • Fluorescence imaging support: To add support for fluorescence reconstruction (fDOT), you need to add a the following option to your configure command:

    --enable-fdot

    Note that the fluorescence module includes code for projecting source distributions onto a mesh surface, and exitance measurements from the mesh surface to an imaging system. The projection code requires the libraries and header files of the Mesa 3D Graphics Library (http://www.mesa3d.org/) installed on your system (tested with version 6.5.3). If Mesa is not installed in a standard system location, you can specify the directories with the --with-mesa=<path>, --with-mesa-includedir=<path> and --with-mesa-libdir=<path> configuration options. You may also have to adapt your LD_LIBRARY_PATH and LD_RUN_PATH environment variables accordingly.

After configuring, set the Toast++ environment by running

source toastenv.csh   # for csh, or
source toastenv.sh    # for bash

Compilation

Core libraries

To build the Toast++ libraries, including the 3rd-party dependencies, type

make

Matlab bindings

After building the core libraries, you can also compile the mex files for the Matlab bindings. This requires a bit of setting up: Inside Matlab, type

mex -setup

and select the option pertaining to the gcc compiler. This will write or update a file

$HOME/.matlab/<matlab-version>/mexopts.sh

Edit this file by appending the line

. $TOASTVER/mexopts.incl

to the end of it (don't forget the single dot at the beginning of the line). This will tell the mex compiler about Toast-specific libraries and include paths. Note: This only works for Matlab versions up to 2013. Newer versions use a different mechanism for mex configuration, which Toast hasn't been adapted to yet.

Then, in the Toast++ main directory, type

make matlab    # for interface v1, and/or
make matlab2   # for interface v2

Once the mex files have been built, from inside Matlab, run the script

mtoast_install    % for interface v1, or
mtoast2_install   % for interface v2

This will open the Matlab path tool and allow you to save the paths for the Toast++ toolbox, so Matlab can locate them in future sessions. If the script finds Toast paths already present in the path list (e.g. from a previous installation) you should delete them when prompted. Note that if you don't have write access to the Matlab installation directories, Matlab will offer to save the paths in a local directory. In that case, you will have to start Matlab from that directory if you want it to load the paths. Otherwise, you will have to run mtoast_install or mtoast2_install again.

Python bindings

To build the Python module, type

make python

Miscellaneous

To bild the source-level documentation, type

make doc

To run the tests, type

make test

Troubleshooting

Fortran compiler problems

The Fortran libraries must be compiled with gfortran, not with g77 which was part of older GNU compiler installations. If you use g77, you will get linker errors related to mismatched underscores in the symbol names exported by the Fortran compilers (this only seems to happen on 64-bit architectures).
