toastMakeMesh

Create a TOAST mesh from vertex and index lists.

Syntax

hMesh = toastMakeMesh (vtx, idx, eltp, [prm])

Parameters

[In] vtxreal [h x 2] or [h x 3]array of vertex positions (2-D or 3-D)
idxinteger [t x v]element index list
eltpinteger [t]element type array
prmreal [h x 1..3](optional) nodal parameter list
[Out] hMeshhandlemesh handle
Dimensions: h = number of nodes (size of h-basis), t = number of elements, v = max. number of nodes per element

Notes

vtx is a double array of dimension h x 2 or h x 3, where h is number of nodes. It contains the node coordinates.

idx is a 1-based integer array of dimension t x v, where t is number of elements, and v is max. number of nodes in an element. It contains the global node indices for all element nodes.

The element index list must be sorted in the order that TOAST expects. (see element header files for local node order).

eltp is an integer array of dimension t x 1. It contains the element type identifiers for all mesh elements. Currently supported element types are:
idelement type
13-noded triangle (old format, deprecated)
34-noded tetrahedron
46-noded wedge
58-noded regular voxel
66-noded triangle
710-noded tetrahedron
86-noded isoparametric triangle
910-noded triangle
1010-noded isoparametric triangle
1110-noded isoparametric tetrahedron
144-noded regular pixel
153-noded triangle

prm is an optional double array of dimension n x 1..3, containing nodal values of absorption (column 1), scatter (column 2) and refractive index (column 3). If prm is not provided, or if it contains less than 3 columns, dummy values are inserted for the missing parameters.

toastMakeMesh labels boundary nodes according to their mesh connectivity. To apply different boundary flags, and to add internal boundaries, use the toastMarkMeshBoundary function.
